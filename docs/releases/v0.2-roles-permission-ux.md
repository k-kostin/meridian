# v0.2 Basic Roles Permission UX

Date: 2026-06-04

## Purpose

Add simple role boundaries for pilot usage without changing stock calculation rules or requiring login in the current local/demo flow.

## Changes

- Added role profile values: admin, operator, viewer.
- Added role-aware server-side permission checks for write operations.
- Added account and role display in the interface.
- Added a basic login page using Django auth.
- Kept anonymous local access compatible with the existing MVP and desktop demo flow.

## Role Matrix

- Admin can manage references, stock operations, inventory operations, demo reset, reports, and exports.
- Operator can create, edit, and post stock documents and inventories, but cannot edit reference dictionaries or reset demo data.
- Viewer can read lists, details, reports, and exports, but cannot create, edit, post, or reset data.
- Anonymous local mode keeps current full-capability MVP behavior for demo and desktop compatibility.

## Deferred Scope

This is not a full enterprise permission system. Object-level permissions, workspaces, invitations, password reset, and audit attribution remain future work.

## Do Not Regress

This version must preserve the `v0.1 MVP Baseline` behavior documented in `docs/releases/v0.1-mvp-baseline.md`.
