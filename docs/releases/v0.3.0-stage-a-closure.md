The current release keeps the original MVP behavior and adds:

- basic role-aware UX and server-side write guards;
- operational activity history for posted documents and inventories;
- built-in quick filters on movement documents and current balances;
- Excel nomenclature import preview and commit flow for new items;
- opening stock import through draft full inventory documents;
- public readiness checks and Stage A audit documentation.

## Do Not Regress

This version must preserve the `v0.1 MVP Baseline` behavior documented in `docs/releases/v0.1-mvp-baseline.md`.

Stage B work must also preserve the Stage A audit result: no direct stock writes from import flows, no bypass of document/inventory posting rules, and no public repository hygiene regressions.
