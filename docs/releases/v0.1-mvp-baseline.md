# v0.1 MVP Baseline

Date: 2026-06-03

## Product Scope

Warehouse Control Desk v0.1 is a working MVP for a small warehouse operation.

This baseline preserves the current useful product, demo, documentation, and desktop preparation work. It is not a rewrite checkpoint and not a claim that the product is production-complete.

## Included Flows

- Units, warehouses, and item catalog.
- Receipt, issue, adjustment, and transfer documents.
- Draft editing for movement and inventory documents before posting.
- Inventory documents with automatic adjustment.
- Current balances with consolidated and by-warehouse presentation.
- Daily, monthly, and period analytics.
- Excel exports for core entities and reports.
- Demo mode with explicit reset.
- `/healthz/` readiness endpoint.
- Electron + Python sidecar scaffold for Windows desktop delivery.
- Tauri documentation retained as an experimental path.
- pywebview retained as fallback/prototype path.

## Do Not Regress

- Posted document lines affect calculated balances.
- Draft documents do not affect balances.
- Issue documents cannot drive stock below zero.
- Transfer documents decrease source warehouse stock and increase destination warehouse stock.
- Inventory does not write balances directly; it creates adjustment documents.
- Full inventory can zero missing positions through automatic adjustment.
- Excel exports remain readable `.xlsx` files with stable download filenames.
- Period analytics keep factual opening, incoming, outgoing, net, and closing values separate.
- Totals never mix different units into one number.
- `/healthz/` remains available for desktop shell readiness.
- Demo mode reset remains explicit and user-confirmed.
- Desktop shell work must not duplicate Django domain rules.

## Known Limitations

- No roles or authentication for real operators.
- No Excel import.
- No audit/activity timeline.
- No saved views or filter presets.
- No counterparty layer.
- No future commitment planning.
- No global search.
- No categories or groups for nomenclature.
- Electron installer has not yet been validated on a clean Windows machine.
- The project directory currently is not a Git repository, so commits/tags cannot be created in this workspace state.

## Open-Source Publication Boundary

The MVP can be prepared for a public GitHub repository, but publication should happen only after a separate repository hygiene pass.

Do not publish these local artifacts:

- `.venv/`
- `db.sqlite3`
- `.playwright-cli/`
- `build/`
- `dist/`
- `desktop/electron_shell/node_modules/`
- `desktop/electron_shell/dist/`
- generated desktop sidecar bundles.

Before opening the repository:

- choose a license;
- add a concise public-facing project description;
- keep demo data synthetic;
- verify no local database, build output, machine paths, or third-party copied repositories are committed;
- run the full verifier stack.
