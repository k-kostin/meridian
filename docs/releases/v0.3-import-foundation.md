# v0.3 Import Foundation

Purpose: add a safe parser-only foundation for future Excel import workflows.

## What Changed

- Added `warehouse_app/imports.py` for reading nomenclature import workbooks.
- Supported a `–Ě–ĺ–ľ–Ķ–Ĺ–ļ–Ľ–į—ā—É—Ä–į` sheet, with fallback to the active sheet.
- Parsed preview rows for `–ź—Ä—ā–ł–ļ—É–Ľ`, `–Ě–į–ł–ľ–Ķ–Ĺ–ĺ–≤–į–Ĺ–ł–Ķ`, `–ē–ī–ł–Ĺ–ł—Ü–į`, `–ź–ļ—ā–ł–≤–Ĺ–į`, and `–ö–ĺ–ľ–ľ–Ķ–Ĺ—ā–į—Ä–ł–Ļ`.
- Added row-level validation errors for missing required fields.
- Added focused parser tests for valid rows, invalid rows, and empty-row skipping.

## Deferred

- Import preview UI.
- Database commit flow for imported nomenclature.
- Opening balance import.
- User-defined import mappings.

## Do Not Regress

- Import parsing must not create or update database records.
- Opening balances must not be written directly as stored balances.
- Future opening balance import must create inventory documents and use existing posting rules.
