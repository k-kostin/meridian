- инвентаризация с автоматической корректировкой;
- первичная настройка через стартовую полную инвентаризацию;
- demo mode с расширенным демонстрационным seed-набором;
- demo mode с явной перезагрузкой набора и confirm-предупреждением;
- защита admin от редактирования проведенных документов;
- retry-based генерация номеров документов и инвентаризаций с опорой на unique constraint;

- текущие остатки;

- опция показа нулевых позиций на текущих остатках и в их Excel-выгрузке;
- отдельный подневный отчет `Дни` с плоской таблицей `дата × позиция`;

- отдельные листы по каждому складу в Excel-выгрузке аналитики при режиме `По складам`;

- наследование выбранного диапазона дат в Excel-выгрузке движений;
- dashboard показывает стартовую инвентаризацию только до начала реальной работы, а затем предлагает обычную инвентаризацию;
- карточки черновиков на dashboard кликабельны и ведут сразу в отфильтрованные списки;
- проведение документа и инвентаризации требует подтверждения пользователя;
- операционная история событий на деталях документов движения и инвентаризаций;
- из журнала документов можно сразу создать приход, расход, корректировку и перемещение;
- черновики документов и инвентаризаций можно редактировать до проведения;
- типы документов в журнале выделены визуальными badge-маркерами;
- sidebar сгруппирован по разделам: обзор, справочники, документы, отчеты;
- бренд в sidebar кликабелен и возвращает на dashboard;
- в подневном и помесячном отчетах есть быстрый переход к предыдущему и следующему периоду;
- в форме документа подсказка по знаку количества перенесена прямо к заголовку колонки;
- empty-state экраны в справочниках и журналах подсказывают следующее действие;
- success-сообщения автоматически скрываются, а error-сообщения остаются на экране;
- Excel-выгрузки по основным сущностям и аналитике;
- отдельные Excel-выгрузки для подневного и помесячного отчетов;
- корректный download filename для `xlsx`-выгрузок через `Content-Disposition`;
- Excel export services без зависимости от HTTP: views формируют `HttpResponse`, сервисы возвращают workbook payload;
- служебный лист `Параметры` в каждой Excel-выгрузке;
- точность отображения количества на уровне единицы измерения.

- DB-level уникальность строк по позиции защищена в документах движения и инвентаризациях;
- перемещение между складами корректно влияет на остатки, отчеты и Excel-выгрузку движений;
- версия приложения `v0.3.0` видна в интерфейсе и синхронизирована с Electron package version;
- базовая role-aware UX-модель для авторизованных пользователей:
  - `admin` может выполнять все действия;
  - `operator` может выполнять складские операции, но не редактировать справочники и не перезагружать демо-данные;
  - `viewer` может читать списки, карточки, отчеты и Excel-выгрузки без write-действий;
  - анонимный локальный режим сохраняет текущий demo/desktop MVP-flow;
- parser-only foundation для импорта Excel-номенклатуры:
  - чтение листа `Номенклатура` или активного листа;
  - preview-строки по `Артикул / Наименование / Единица / Активна / Комментарий`;
  - row-level validation errors для обязательных полей;
  - без записи в БД и без импорта остатков;
- экран предпросмотра импорта номенклатуры:
  - загрузка `.xlsx`;
  - показ preview-строк и row-level validation errors;
  - доступ только для пользователей с правом управления справочниками;
  - commit-flow для создания новых позиций после проверки всего файла;
  - блокировка неизвестных единиц, существующих артикулов и дублей артикулов в файле;
- импорт стартовых остатков:
  - загрузка `.xlsx` с листом `Стартовые остатки` или активным листом;
  - показ preview-строк и row-level validation errors;
  - commit-flow создает черновик полной инвентаризации для одного склада;
  - прямой записи остатков и автоматического проведения нет;

## Известные ограничения

- нет обновления существующих позиций через импорт Excel;
- нет автоматического создания единиц измерения через импорт Excel;
- нет расширенного импорта стартовых остатков для нескольких складов в одном файле;
- нет разреза по группам номенклатуры;
- Windows desktop-сборка еще не собрана и не проверена на чистой машине.
- Electron shell scaffold добавлен и проверен в dev-smoke на macOS, но еще не проверен через Windows NSIS installer.
- Tauri shell пока зафиксирован как структура и экспериментальная стратегия, но не развернут как готовый toolchain-проект.
- пользовательские сохраненные представления с именами, владельцем и правами доступа пока не реализованы; текущие presets являются только встроенными быстрыми фильтрами.
- enterprise RBAC не реализован: нет object-level permissions, приглашений, password reset и user attribution в операционной истории.
